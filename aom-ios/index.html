Testing iOS 12.2's implementation of Accessibility Events and the Accessibility Object Model (AOM)

This only works if you have Accessibility Events and Accessibility Object Model support turned on in your iOS settings.

First steps:

  1. Read up on why screen reader detection is problematic. Seriously, do this first.
  2. Make sure that you are running iOS 12.2 or later, and that Accessibility Events are enabled. (Settings > General > Accessibility > VoiceOver > Web)
  3. Make sure that the Accessibility Object Model implementation is enabled. (Settings > Safari > Advanced > Experimental Features)

Note: The examples here are meant for testing purposes only. While I can pretty much guarantee that any ARIA used conforms to the WAI-ARIA specification, I haven't added click, touch, or keyboard events to all of the examples, as my purpose here is to isolate the functionality of Apple's AOM implementation.
